Срочный фикс ошибки 400 Bad Request при upsert profiles и insert messages!

ПРОБЛЕМА:
Ты передаешь в supabase.from('profiles').upsert() или в messages лишние/несуществующие поля или полный объект user из auth, из-за чего Supabase REST API выдает 400 (Bad Request).

ЖЁСТКОЕ ТРЕБОВАНИЕ К КОДУ ВО ВСЕХ ФАЙЛАХ (auth.js, ui.js, chat.js, app.js):

1. ПРИ СОХРАНЕНИИ ПРОФИЛЯ (profiles.upsert):
ВСЕГДА создавай строго очищенный объект ТОЛЬКО из 5 полей. Категорически ЗАПРЕЩЕНО передавать весь объект `user`!

Напиши ровно так:
const cleanProfileData = {
  id: userId,
  username: usernameValue || 'User',
  avatar_url: avatarUrlValue || null,
  bio: bioValue || '',
  status: 'online'
};
const { error } = await supabase.from('profiles').upsert(cleanProfileData, { onConflict: 'id' });

2. ПРИ ОТПРАВКЕ СООБЩЕНИЯ (messages.insert):
Передавай ТОЛЬКО эти 3 поля:
const cleanMessageData = {
  sender_id: currentUserId,
  receiver_id: targetUserId,
  content: messageText.trim()
};
const { error } = await supabase.from('messages').insert([cleanMessageData]);

3. ОБЕРНИ КАЖДЫЙ ВЫЗОВ В try...catch:
Если вызов возвращает error, выводи console.warn(error), но НЕ ОСТАНАВЛИВАЙ выполнение скриптов и отрисовку UI!

Исправь и перепиши файлы:
- js/auth.js
- js/ui.js
- js/chat.js
- js/app.js
