Развёртывание и быстрая настройка SENTCOR

1) Настройка конфигурации
- Скопируйте файл `js/config.js.example` в `js/config.js` и заполните поля SUPABASE_URL и SUPABASE_ANON_KEY и LIVEKIT_URL.
- Не храните реальные секреты в публичном репозитории. Используйте CI/скрипты для подстановки перед деплоем.

2) Supabase
- В панели Supabase откройте SQL Editor и выполните содержимое `schema.sql` (создастся схема базы, таблицы и триггеры).
- Создайте storage bucket `avatars` (публичный или настроенный для выдачи публичных URL).
- Убедитесь, что RLS включён и политики применены (schema.sql уже содержит политики).

3) Edge Function (LiveKit token)
- Разверните ваш edge function (в `supabase/functions/livekit-token`) в Supabase Functions.
- Передайте LIVEKIT API KEY/SECRET как секреты функции.

4) GitHub Pages
- Вы можете публиковать `index.html` как статический сайт через GitHub Pages (ветка `main`, папка root) или использовать статический хостинг.

5) Проверка
- Откройте сайт, выполните регистрацию и проверьте: профиль создаётся, можно загрузить аватар (bucket `avatars`), дневные логины добавляются.

Если нужны — я могу помочь с автоматическим развёртыванием функций и политик через скрипты или CI.
